This document outlines all content needed before and during website development. Use this as a checklist to gather required materials.

## Content Submission Guidelines

### File Naming Conventions
Use clear, descriptive file names:
- No spaces (use hyphens)
- Include dates in YYYY-MM-DD format
- Use lowercase
- Include version numbers if multiple drafts

**Examples:**
- `2025-01-board-meeting-minutes.pdf`
- `adopted-budget-fy2025.pdf`
- `wildfire-preparedness-guide.pdf`
- `commissioner-jane-smith-photo.jpg`

### File Formats

#### Documents
- **Preferred:** PDF (searchable text, not scanned images)
- Size: Under 10MB per file (compress if larger)
- Ensure PDFs are accessible (tagged, searchable)

#### Images
- **Logos:** SVG (preferred) or PNG with transparency
- **Photos:** JPG format, high quality
- **Graphics:** PNG for graphics with text/transparency
- Size: At least 2000px on longest side for photos
- File size: Under 5MB (compress if needed)

#### Videos
- Host on YouTube or Vimeo (preferred)
- Or provide MP4 format, H.264 codec
- Include captions/transcripts

### Text Content
- Provide in Microsoft Word, Google Docs, or plain text
- Use proper heading structure
- Include alt text descriptions for images
- Indicate any links that should be included
- Proofread for spelling and grammar

### Metadata to Include
For each document, provide:
- Document title
- Date (created/adopted/approved)
- Brief description
- Author/department (if applicable)
- Any keywords for search

## Content Review Process

### Before Submission
- [ ] Proofread all text content
- [ ] Check all dates and facts
- [ ] Verify all contact information is current
- [ ] Ensure all people have approved their photos/bios
- [ ] Get necessary approvals from leadership
- [ ] Verify you have rights to use all content
- [ ] Check file formats and sizes

### During Development
- [ ] Review content as it's added to website
- [ ] Provide feedback on layout and design
- [ ] Verify accuracy of information
- [ ] Test all links and downloads
- [ ] Approve final content before launch

### After Launch
- [ ] Establish content update schedule
- [ ] Designate content owners for each section
- [ ] Set up reminder system for regular updates
- [ ] Plan for ongoing content needs
